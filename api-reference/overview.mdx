# Shop API Overview

Complete REST API for integrating with the Loyalty.lt loyalty platform. Use this API to manage loyalty cards, process transactions, and handle customer authentication.

## Base URLs

| Environment | URL |
|-------------|-----|
| **Production** | `https://api.loyalty.lt/{locale}/shop/` |
| **Staging** | `https://staging-api.loyalty.lt/{locale}/shop/` |

<Note>
**Locale** parameter: `lt` (Lithuanian) or `en` (English)
</Note>

## Authentication

All Shop API endpoints require authentication via HTTP headers:

```bash
curl -X GET 'https://api.loyalty.lt/lt/shop/shops' \
  -H 'X-API-Key: lty_your_api_key' \
  -H 'X-API-Secret: your_api_secret' \
  -H 'Content-Type: application/json'
```

| Header | Description |
|--------|-------------|
| `X-API-Key` | Your API key (starts with `lty_`) |
| `X-API-Secret` | Your API secret |
| `Content-Type` | `application/json` |

<Tip>
Get your API keys from [Partners Portal](https://partners.loyalty.lt) â†’ API Credentials section.
</Tip>

## Response Format

All responses follow a consistent JSON structure:

### Success Response

```json
{
  "success": true,
  "code": 200,
  "request_id": "abc-123-def",
  "data": { ... }
}
```

### Paginated Response

```json
{
  "success": true,
  "code": 200,
  "request_id": "abc-123-def",
  "meta": {
    "current_page": 1,
    "last_page": 5,
    "per_page": 15,
    "total": 73
  },
  "data": [ ... ]
}
```

### Error Response

```json
{
  "success": false,
  "code": 400,
  "message": "Validation error",
  "errors": {
    "field_name": ["Error message"]
  }
}
```

## HTTP Status Codes

| Code | Description |
|------|-------------|
| `200` | Success |
| `201` | Created |
| `400` | Bad Request / Validation Error |
| `401` | Unauthorized - Invalid API credentials |
| `403` | Forbidden - Access denied |
| `404` | Not Found |
| `422` | Unprocessable Entity |
| `429` | Too Many Requests - Rate limited |
| `500` | Server Error |

## Rate Limits

| Tier | Requests/Hour | Requests/Minute |
|------|---------------|-----------------|
| Standard | 1,000 | 60 |
| Premium | 10,000 | 300 |

Rate limit headers are included in all responses:

```
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 847
X-RateLimit-Reset: 1703707200
```

<Warning>
Exceeding rate limits will result in `429 Too Many Requests` responses. Contact support for higher limits.
</Warning>
