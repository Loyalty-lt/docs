# Loyalty.lt Partner API

Welcome to the Loyalty.lt Partner API documentation. This API enables you to integrate loyalty program features into your applications, POS systems, and e-commerce platforms.

## Base URLs

<Tabs>
  <Tab title="Production">
    ```
    https://api.loyalty.lt/{locale}/shop/
    ```
    
    Use for live integrations with real customer data.
  </Tab>
  
  <Tab title="Staging">
    ```
    https://staging-api.loyalty.lt/{locale}/shop/
    ```
    
    Use for development and testing. Test data only.
  </Tab>
</Tabs>

**Locale**: `lt` (Lithuanian) or `en` (English)

## Authentication

All Partner API endpoints require authentication via HTTP headers:

```bash
curl -X GET 'https://api.loyalty.lt/lt/shop/shops' \
  -H 'X-API-Key: lty_your_api_key' \
  -H 'X-API-Secret: your_api_secret' \
  -H 'Content-Type: application/json'
```

| Header | Description |
|--------|-------------|
| `X-API-Key` | Your Partner API Key (starts with `lty_`) |
| `X-API-Secret` | Your Partner API Secret |
| `Content-Type` | `application/json` |

## Response Format

All responses follow a consistent JSON structure:

### Success Response

```json
{
  "success": true,
  "code": 200,
  "request_id": "abc-123-def",
  "data": { ... }
}
```

### Paginated Response

```json
{
  "success": true,
  "code": 200,
  "request_id": "abc-123-def",
  "meta": {
    "current_page": 1,
    "last_page": 5,
    "per_page": 15,
    "total": 73
  },
  "data": [ ... ]
}
```

### Error Response

```json
{
  "success": false,
  "code": 400,
  "message": "Validation error",
  "errors": {
    "card_id": ["The card_id field is required."]
  }
}
```
